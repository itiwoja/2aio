2AIO の「既に動いているプロジェクトに機能を足す・バグを直す」モード。ユーザーは対象 repo と
やりたいこと（粒度不問。1行の思いつきでも、再現手順付きのバグ報告でもよい）を渡すだけでよい。

## 手順

1. **入力確認**: 第1引数（repoパス）が空なら「対象のリポジトリはどこですか？」と聞く。
   第2引数（機能記述、または `--fix` バグ報告）が空なら「何を実装・修正しますか？」と1問だけ聞く。
2. **レーン実行**: `~/.claude/2aio/lanes/2aio-dev.md` を Read し、その指示に $ARGUMENTS を厳密に従って実行する。
   - `--fix` 無し → feature モード（機能追加。曖昧な思いつきでもレーン内で軽量 spec 化されるので OK）
   - `--fix` あり → fix モード（バグ修正。再現確認 → 失敗する回帰テスト → 最小修正の順）
3. **報告**: 完了後、ブランチ名・コミット・変更ファイル・QA結果・（`--pr` 指定時のみ）PR URL を、
   レーン内部の用語を使わずに報告する。

## 禁止事項
- ユーザーにレーン内の Phase 名や内部用語を意識させない。
- レーン定義に無い工程を足さない（取締役会・PRD・リサーチを復活させない）。
- fix モードで「ついでに直す」を許可しない（レーン側のスコープ防衛に従う）。
