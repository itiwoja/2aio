# mO.ai TUI — 画面の側

neofetch 風のレイアウト。左にモアイの点描、右にそのエージェントの情報。
**側（見た目と画面遷移）だけを作っています。** 実行系・モデル切り替え・認証はどこにも繋がっていません。

```
cd tui
npm install
npm start            # 対話（要TTY）
npx tsx preview.jsx  # 全5ページを一度に描き出す静的プレビュー
```

ルート (`2aio/`) の依存ゼロ方針を壊さないよう、TUIは独立パッケージとして `tui/` に閉じてあります。

## ページ

`↑/↓` で送る5ページ。`←/→` の「項目」はページによって指すものが変わります（moai/dashboard では像、system では黒曜石、settings では設定行）。

| # | ページ | 中身 |
|---|---|---|
| 1 | dashboard | 25体をレーン別に4カラムで一望。どの像に黒曜石が入っているか |
| 2 | system | サブスク / APIキー / ローカル の3系統を1本のリストで選ばせる |
| 3 | moai | 像＋そのエージェントの実像（neofetch風）。既定のページ |
| 4 | logs | 起動中の像と時系列ログ |
| 5 | settings | 設定の側だけ |

## モアイの点描

`components/Moai.jsx` の `RAW`。画像ファイルは読み込みません。

点字（U+2800〜）を使っています。1セルに横2×縦4の点が入り、文字セルの縦長（横1:縦2）をちょうど打ち消すので、点が正方格子に並びます。

> **フォント要件**: 点字を持つ等幅フォントが必要です（Cascadia Code / DejaVu Sans Mono など）。
> また点字は East Asian Width が **Ambiguous** です。端末の「あいまいな文字幅」を*全角*に設定していると
> 1文字が2セル幅で描かれ、レイアウトが崩れます。*半角*にしてください。

### 点の取り込み方（ハマりどころ）

参照モックに描かれていた点を写し取っています。手順は、点描領域の明画素を**連結成分に分けて、各塊の重心を点字1点に落とす**（頭部で1105点）。

「セル内に明画素があるか」で判定すると**全部潰れます**。点は5〜7pxあってアンチエイリアスの滲みを伴うため、1つの点が最大4セルを点灯させ、隣同士がくっついて `⣿` の塊になります。重心を取るのが要点です。

なお元のモアイ写真（暗い像＋暗いハロー）から起こす方法は試して捨てました。硬い輪郭を持たないため、ほぼ全画面まで拡大しても黒い塊にしかなりません。輪郭を座標で定義して打ち直す方法も試しましたが、こちらは収束しませんでした。

### 大きさ

全身(1:2.4)ではなく**頭部(1:1.6)で切って** `28 × 23`。縦を詰めるとき全身のまま縮めるのは誤りで、セル数だけ減って点の密度が上がり、輪郭が黒い塊になります。切る方が正しい。

なお moai ページの高さを決めているのは絵ではなく**右パネル**です。絵だけ縮めても縦は縮みません。

## moai ページに何を映すか

**マシンの情報（ホスト名・シェル・カーネル・IP・CPU・メモリ）は出しません。** エージェントは機械ではなく、
それらは全部ユーザー側のマシンの値です。実データに繋ぐ段になって埋められない項目を最初から置かない、
という方針です（初期案では `engineer.moai` のようなホスト名を捏造していました。それが症状でした）。

代わりに、エージェントが実際に持っていて見る価値があるものだけを置いています。

| 項目 | なぜ |
|---|---|
| 黒曜石 / 系統 | どのモデルで動くか。系統（サブスク/APIキー/ローカル）で制約が変わる |
| クォータ | サブスク型の核心的制約。並列実行数の判断に直結する |
| いまのタスク・進捗・自己修正回数 | engineer は自己修正3回で止まりエスカレーションするため、残り回数が予兆になる |
| 待っているもの | 走っていない像の主役。下記 |
| 直近の実行 | 走っていない像でも画面が死なないように |
| 呼び出し関係 | 誰から起動され次に誰へ渡すか。オーケストレーションの流れが見える |
| モデル固定の有無 | 実装トリオ(engineer/qa/devops)は sonnet 固定でセッション継承しない（ARCHITECTURE 原則4） |

### 待ち条件はゲートそのもの

25体のうち走っているのは常に数体で、残りは待機です。その像に「いまのタスク」はないので、代わりに
**何を待っているか**を出します。ここは飾りではなく `ARCHITECTURE.md` に実在するゲートを写しています。

- **2aio-qa** — `WBS完走 → 受け入れ条件の検証` / 状況 `T-014 実行中 · 14/18`
- **2aio-devops** — `QAパス + deploy_approved: true`（原則2: 承認は state.md のみ）
- **2aio-release-manager** — `devops の Step 2.5 通過`（原則3: セキュリティゲートは devops Step 2.5 が唯一の正）
- **レビュー系** — `QAパス後の変更ファイル一覧`（原則3.5: コード品質レビューは QAパス後のみ）

黒曜石が入っていない像には `条件が揃っても動けない（黒曜石なし）` を赤で足しています。ゲートの待ちと
黒曜石の不在は別の理由で止まっているので、混ぜないための一行です。

### クォータは像ではなく黒曜石に付く

同じプロバイダの黒曜石をはめた像は**全部で1つの窓を食い合います**。つまり「この像のクォータ」というものは
存在しません。だからパネルには `5時間窓 · 共有 10体 · 対話込み` と、**その窓を共有している像の数**を併記して
います。ここを出さないと並列実行数の判断ができません。「対話込み」は、headless 実行が対話セッションと同じ
窓を共有するという調査結果を指しています。

系統によって性質が違うので、表示も分けています。

- **サブスク型** = 5時間ローリング窓。残量ゲージ（20%未満で赤、40%未満で橙）
- **APIキー型** = 従量課金。窓がないので「従量課金 — 窓なし」
- **ローカル型** = 自前ホスト。「上限なし」
